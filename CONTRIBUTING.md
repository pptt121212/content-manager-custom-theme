# 贡献指南

欢迎社区贡献，但请确保所有贡献也遵循非商业使用限制。

## 贡献方式

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

## 开发规范

### 代码风格
- 遵循 WordPress 编码标准
- 使用清晰的变量和函数命名
- 添加适当的注释说明复杂逻辑
- 保持代码简洁易读

### 提交信息
- 使用清晰、有意义的提交信息
- 遵循常规的提交格式：`<type>: <subject>`
- 示例：
  - `feat: 添加新功能`
  - `fix: 修复bug`
  - `docs: 更新文档`
  - `style: 代码格式调整`
  - `refactor: 代码重构`
  - `test: 添加测试`
  - `chore: 构建过程或辅助工具的变动`
